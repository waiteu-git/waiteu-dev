# waiteu.dev トップページ再設計

日付: 2026-07-10
対象: `index.html`（単一ファイル）

## 背景と目的

現在のトップページは hero・カード2枚・リンク行・フッターで構成され、ほぼスクロールが発生しない。2つの課題がある。

1. **カードから各サービスへ辿り着きにくい。** カード内の CTA テキストリンクだけがクリック可能で、カード本体は反応しない。
2. **分量が足りない。** waiteu という個人開発者が何者で、何を作り、どこへ向かうのかがトップから読み取れない。

本設計は、トップページを「各サービスのランディングへ送り出す、束ねるページ」として作り直す。各サービスの詳細（機能一覧・スクリーンショット・FAQ・インストール手順）は既存のランディングが担っており、トップで重複させない。

## 非目標

- ビルド工程の導入（現状の静的HTML1枚・Cloudflare Pages 配信を維持する）
- claude.ai/design のデザインシステム連携（同期する実体となるコンポーネントライブラリが存在しないため、今回は見送る）
- `lms.waiteu.dev` および `lms.waiteu.dev/app` 側の変更
- 配色・フォントの刷新（後述のとおり維持する）

## リンク先の確定

| カード | リンク先 |
|---|---|
| リタス（Litus） | `https://lms.waiteu.dev/app` |
| LETUS課題ウォッチャー | `https://lms.waiteu.dev/` |

**Chrome Web Store への直リンクは廃止する。** カード全体を `<a>` にする以上、その内側にもう一つ `<a>` は置けない（入れ子リンクはHTML的に不正で、キーボード操作とスクリーンリーダーが壊れる）。`lms.waiteu.dev` にインストールボタンが存在するため導線は失われない。トップから直接ストアへ送ると、何の拡張か理解しないまま Chrome Web Store に着地することになり、かえって離脱する。

単一カラムの縦スクロール。上から順に読ませる。

## 文言（確定稿）

### Hero

> waiteu
>
> 東京理科大生。学生のためのツールを、個人でつくっています。

### About — 動機

> 課題の提出期限を、一度落としかけたことがある。
>
> LETUSを開いて履修科目をひとつずつ辿り、締切を目で拾う。必要な情報がLETUSとCLASSに分かれて置かれている。使いづらいものを、気合いで使い続けるしかないのはおかしい。
>
> だから自分で作ることにした。まずブラウザ拡張を、次にスマートフォンアプリを。

第1段落は個人の失敗、第2段落は構造の問題、第3段落は行動。視野が個人から構造へ開いていく順序になっている。第2段落末尾の「おかしい」が向いている先は LETUS と CLASS であって、我慢して使い続けている学生ではない。ここを「賢い選択ではない」等に書き換えると、批判の矛先が読み手自身に向く。

### About — スタンス

> **無料で始める。** 学生からお金を取るために作っていないが、安全に、止めずに提供し続けるにはお金がかかる。その負担が無視できなくなったときは、有料化して支援をお願いすることがある。黙って始めることはしない。
>
> **パスワードは読み取らない。** ログイン情報に触れない設計にしている。
>
> **動かし続けることを優先する。** 機能を足すより壊れないことを選ぶ。学期の途中で止まらないことが、いちばんの機能だと思っている。

3つのスタンスは「お金 → 安全 → 継続」と軸が分かれている。

1つ目は *free-first* であって *free-forever* ではない（`litus/docs/2026-07-04-free-first-strategy-design.md` に基づく）。「永久に無料」と書いてはならない。守れるかどうかが自分の意思の外にあるためである。代わりに「黙って始めることはしない」と書く。これは自分の意思だけで守れる約束であり、先に条件を明示しておけば、将来の有料化は約束の違反ではなく履行になる。

「黙って始めることはしない」は短文のまま維持する。4文かけて事情を説明したあとの言い切りであり、前の文に繋ぐと約束として響かなくなる。

### About — 献辞

> 日々を生き抜く、すべての理科大生へ。

説明ではなく宛名である。助詞は「に」ではなく「へ」。「に」は対象を指して説明文の続きになるが、「へ」は差し出す方向を示すため宛名として立つ。読点は「日々を生き抜く」を一拍置いて受け止めさせるために置く。

キャンパス（神楽坂・野田・葛飾）で絞らない。大学の外へも広げない。現時点では理科大に閉じる判断であり、他大学展開を示唆する含みを持たせない。

## UI

### 維持するもの

- 配色。ライト `--bg: #f7f4ef` / `--fg: #2b2723` / `--accent: #d9724c`、ダーク `--bg: #221e1a` / `--fg: #f2ede6` / `--accent: #e8875f`
- フォント。Noto Sans JP（Google Fonts、現行の `<link>` をそのまま使う）
- `prefers-color-scheme` によるダークモード対応
- 既存のCSS変数名

各サービスのランディングはそれぞれ独自のテーマを持つ（拡張は現行配色、リタスは翠）。トップはそれらを束ねる中立の色として立つため、どちらかに寄せない。

### 変更するもの

- **レイアウト。** 中央寄せ・1画面完結をやめ、左寄せ単一カラム（本文 `max-width` 約 680px）の縦スクロールにする。長文が読める行長を確保する。
- **カード。** `<article>` から `<a class="card">` へ。hover で `translateY(-2px)`、topbar が伸びる。`:focus-visible` でフォーカスリングを明示する。状態を示す pill（「事前登録受付中」「公開中」）を追加する。
- **セクション見出し。** 既存の `.eyebrow`（小さいラベル）と `.rule`（オレンジの短い罫）を流用する。新しい部品を増やさず、既にある語彙で組む。
- **献辞。** 本文から離し、中央寄せ・やや大きめ・上にオレンジの罫を一本。文章の続きではなく区切りとして見せる。
- **ロードマップ。** 縦の罫にドットを打つタイムライン。フェーズごとに状態 pill を置く。
- **Contact。** 現行の `.links`（連絡先が3つ並ぶだけの帯）を、見出しと呼びかけ文を持つセクションに昇格する。

### アニメーション

現行の `fadeInUp` は hero とファーストビューぶんだけ残す。下部セクションは `IntersectionObserver` でスクロール時に発火させる。JS はこの用途のみで、数行に収める。

`@media (prefers-reduced-motion: reduce)` を追加し、すべてのアニメーションを無効化する。**現行ページはこれに未対応であり、本改修で修正する。**

## 技術

- 成果物は `index.html` 1ファイル。CSS は `<style>` に、JS は `<script>` にインラインで置く（現行方式を踏襲）
- 外部依存は Google Fonts のみ（現行どおり）
- 配信は変更なし。`master` への push で GitHub Actions が `wrangler pages deploy` を実行する
- `.assetsignore` に `docs` を追加する。現行の除外は `.github` / `.git` / `.assetsignore` / `README.md` のみで、`docs/` を除外しないと本設計ドキュメントが `waiteu.dev/docs/...` として公開される（`litus/terms.html` が配信されているのと同じ仕組み）
