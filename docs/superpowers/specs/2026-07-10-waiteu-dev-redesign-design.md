# waiteu.dev トップページ再設計

日付: 2026-07-10
対象: `index.html`（単一ファイル）

## 背景と目的

現在のトップページは hero・カード2枚・リンク行・フッターで構成され、ほぼスクロールが発生しない。2つの課題がある。

1. **カードから各サービスへ辿り着きにくい。** カード内の CTA テキストリンクだけがクリック可能で、カード本体は反応しない。
2. **分量が足りない。** waiteu という個人開発者が何者で、何を作り、どこへ向かうのかがトップから読み取れない。

本設計は、トップページを「各サービスのランディングへ送り出す、束ねるページ」として作り直す。各サービスの詳細（機能一覧・スクリーンショット・FAQ・インストール手順）は既存のランディングが担っており、トップで重複させない。

## 非目標

- ビルド工程の導入（現状の静的HTML1枚・Cloudflare Pages 配信を維持する）
- claude.ai/design のデザインシステム連携（同期する実体となるコンポーネントライブラリが存在しないため、今回は見送る）
- `lms.waiteu.dev` および `lms.waiteu.dev/app` 側の変更
- 配色・フォントの刷新（後述のとおり維持する）

## リンク先の確定

| カード | リンク先 |
|---|---|
| リタス（Litus） | `https://lms.waiteu.dev/app` |
| LETUS課題ウォッチャー | `https://lms.waiteu.dev/` |

**Chrome Web Store への直リンクは廃止する。** カード全体を `<a>` にする以上、その内側にもう一つ `<a>` は置けない（入れ子リンクはHTML的に不正で、キーボード操作とスクリーンリーダーが壊れる）。`lms.waiteu.dev` にインストールボタンが存在するため導線は失われない。トップから直接ストアへ送ると、何の拡張か理解しないまま Chrome Web Store に着地することになり、かえって離脱する。

単一カラムの縦スクロール。上から順に読ませる。

**文体は全編を敬体（です・ます）で統一する。** 当初は「自分を語る文は常体、読者に向けた文は敬体」という使い分けにしていたが、規則を宣言していないため単なる不統一に見えた。敬体で揃えると書き手と読み手の距離が一定になり、個人ブログではなく説明のためのページとして読める。

**個人の失敗談は書かない。** 当初の「課題の提出期限を、一度落としかけたことがある」という導入は削除した。語りを減らしてポリシーを主役にする以上、最初に立つべきは個人の体験ではなく、誰にでも観測できる不便のほうである。

**Policy の献辞と Contact の呼びかけは、対象範囲が意図的に異なる。** 献辞は「すべての理科大生へ」と限定し、呼びかけは誰も限定しない。作る相手は絞るが、聞く相手は絞らない。この非対称は意図的なものであり、片方に合わせて揃えてはならない。

## 文言（確定稿）

### Hero

> waiteu
>
> 東京理科大生。学生のためのツールを、個人でつくっています。

### About — 概要

> 東京理科大の学生に向けて、LETUSとCLASSを使いやすくするツールをつくっています。必要な情報が2つのシステムに分かれて置かれていて、締切を追うだけで手間がかかります。使いづらいものを、気合いで使い続けるしかないのは、おかしなことです。
>
> だから自分で作ることにしました。まずブラウザ拡張を、次にスマートフォンアプリを。

「おかしなことです」が向いている先は LETUS と CLASS であって、我慢して使い続けている学生ではない。ここを「賢い選択ではありません」等に書き換えると、批判の矛先が読み手自身に向く。

第2段落は Products セクションへの橋である。削らないこと。

### Policy — 開発ポリシー

> **無料で始めます。** 学生からお金を取るために作っていませんが、安全に、止めずに提供し続けるにはお金がかかります。その負担が無視できなくなったときは、有料化して支援をお願いすることがあります。黙って始めることはしません。
>
> **パスワードを読み取りません。** ログイン情報に触れない設計にしています。
>
> **必要のないデータを持ちません。** 動作に不要な情報は集めず、集めたものを外部に渡すこともありません。広告は載せず、行動の追跡もしません。
>
> **動かし続けることを優先します。** 機能を足すより、壊れないことを選びます。学期の途中で止まらないことが、いちばんの機能だと思っています。
>
> **公式のサービスを置き換えません。** LETUSとCLASSは、これからも大学の正式な窓口です。ここでつくるものは、そこへ辿り着くまでを短くするだけの道具です。
>
> **何をしているか、説明できる状態にします。** 変更履歴を公開し、権限や取得するデータの範囲を明示します。

軸は「お金 → 安全 → データ → 継続 → 領分 → 透明性」と分かれている。

**1つ目**は *free-first* であって *free-forever* ではない（`litus/docs/2026-07-04-free-first-strategy-design.md` に基づく）。「永久に無料」と書いてはならない。守れるかどうかが自分の意思の外にあるためである。代わりに「黙って始めることはしません」と書く。これは自分の意思だけで守れる約束であり、先に条件を明示しておけば、将来の有料化は約束の違反ではなく履行になる。

**3つ目**は事実確認済みである。2026-07-10 時点で、拡張のソースにも `lms.waiteu.dev` / `lms.waiteu.dev/app` / `waiteu.dev` の配信HTMLにも、Google Analytics その他の解析スクリプトは存在しない。拡張の公開プライバシーポリシーも「抽出データは `chrome.storage.local` にのみ保存され、外部サーバーに送信されない」と明言している。**将来いずれかに解析を導入する場合、この項目を先に書き換えること。** ユーザー数の把握だけが目的なら、Chrome Web Store やアプリストアの開発者ダッシュボードで足りる（拡張やアプリにコードを仕込む必要がない）。

**5つ目**は落とさないこと。大学の非公式ツールを個人が配るという構図に対して、大学側と学生側の双方が抱く不安に先回りして答える唯一の項目である。他の項目が「良いものを作ります」の言い換えであるのに対し、これだけは自分の領分を狭く申告している。

**6つ目**を残す以上、取得するデータの範囲がどこかで実際に明示されている必要がある。拡張は `privacy-policy.md`、リタスは `/litus/terms` が該当する。

「黙って始めることはしません」は短文のまま維持する。3文かけて事情を説明したあとの言い切りであり、前の文に繋ぐと約束として響かなくなる。

### Policy — 献辞

> 日々を生き抜く、すべての理科大生へ。

説明ではなく宛名である。助詞は「に」ではなく「へ」。「に」は対象を指して説明文の続きになるが、「へ」は差し出す方向を示すため宛名として立つ。読点は「日々を生き抜く」を一拍置いて受け止めさせるために置く。

キャンパス（神楽坂・野田・葛飾）で絞らない。大学の外へも広げない。現時点では理科大に閉じる判断であり、他大学展開を示唆する含みを持たせない。

## UI

### 維持するもの

- 配色。ライト `--bg: #f7f4ef` / `--fg: #2b2723` / `--accent: #d9724c`、ダーク `--bg: #221e1a` / `--fg: #f2ede6` / `--accent: #e8875f`

**ただし派生トークンはコントラストのため調整した（実装時に判明）。** 現行サイトのライトモードは WCAG AA（4.5:1）を満たしていなかった。`--muted` 3.94:1、`--muted-strong` 2.97:1、CTA のアクセント文字 2.79:1。再設計で `--muted` の本文（スタンス3項目、ロードマップ説明）が大幅に増えるため、放置できない。

- `--muted` を α .6 → **.68**、`--muted-strong` を α .5 → **.66**、`--footer` を α .45 → **.66**（ライトのみ）
- 文字色専用の `--accent-text` を追加。ライト `#a84726`、ダークは `#e8875f`（現行のまま）。`.cta` と `.pill-live` の文字に使う
- **`--accent: #d9724c` は文字以外（罫、カードのトップバー、タイムラインのドット、pillの枠線）にそのまま残す。** ブランド色は変えていない

実測結果: ライト最小 4.59:1、ダーク最小 4.55:1。両モードとも全テキストが AA 適合。
- フォント。Noto Sans JP（Google Fonts、現行の `<link>` をそのまま使う）
- `prefers-color-scheme` によるダークモード対応
- 既存のCSS変数名

各サービスのランディングはそれぞれ独自のテーマを持つ（拡張は現行配色、リタスは翠）。トップはそれらを束ねる中立の色として立つため、どちらかに寄せない。

### 変更するもの

- **レイアウト。** 中央寄せ・1画面完結をやめ、左寄せ単一カラム（本文 `max-width` 約 680px）の縦スクロールにする。長文が読める行長を確保する。
- **カード。** `<article>` から `<a class="card">` へ。hover で `translateY(-2px)`、topbar が伸びる。`:focus-visible` でフォーカスリングを明示する。状態を示す pill（「事前登録受付中」「公開中」）を追加する。
- **セクション見出し。** 既存の `.eyebrow`（小さいラベル）と `.rule`（オレンジの短い罫）を流用する。新しい部品を増やさず、既にある語彙で組む。
- **献辞。** 本文から離し、中央寄せ・やや大きめ・上にオレンジの罫を一本。文章の続きではなく区切りとして見せる。
- **ロードマップ。** 縦の罫にドットを打つタイムライン。フェーズごとに状態 pill を置く。
- **Contact。** 現行の `.links`（連絡先が3つ並ぶだけの帯）を、見出しと呼びかけ文を持つセクションに昇格する。

### アニメーション

現行の `fadeInUp` は hero とファーストビューぶんだけ残す。下部セクションは `IntersectionObserver` でスクロール時に発火させる。JS はこの用途のみで、数行に収める。

`@media (prefers-reduced-motion: reduce)` を追加し、すべてのアニメーションを無効化する。**現行ページはこれに未対応であり、本改修で修正する。**

## 技術

- 成果物は `index.html` 1ファイル。CSS は `<style>` に、JS は `<script>` にインラインで置く（現行方式を踏襲）
- 外部依存は Google Fonts のみ（現行どおり）
- 配信は変更なし。`master` への push で GitHub Actions が `wrangler pages deploy` を実行する
- `.assetsignore` に `docs` を追加する。現行の除外は `.github` / `.git` / `.assetsignore` / `README.md` のみで、`docs/` を除外しないと本設計ドキュメントが `waiteu.dev/docs/...` として公開される（`litus/terms.html` が配信されているのと同じ仕組み）

### 未検証の項目（環境の制約による）

- [ ] **キーボードの Tab でカードにフォーカスでき、Enter で遷移する。** フォーカスリングが見えることは確認済み（`:focus-visible` でアクセント色の実線アウトライン、`outline-offset` あり）。ただし Tab / Enter の合成キーイベントが検証用ブラウザのページに届かず、キー操作そのものは実測できていない。カードはネイティブの `<a href>` で `keydown` ハンドラも `onclick` も持たないため、Enter による遷移はブラウザの既定動作に委ねられている
