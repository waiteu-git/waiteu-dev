# リタス 緊急停止（kill switch）操作手順書

**これは何**: リタス（アプリ）を遠隔で止めるスイッチ。`https://waiteu.dev/litus/status.json` を書き換えて `master` にpushするだけで、全端末のアプリが停止する。大学からの要請時など「24時間以内に止める」ための手段。

**いつ使う**: 大学からの停止要請／重大な不具合・脆弱性の発見時。迷ったら広めに（＝`all`で）止める。

**このファイルは公開されない**（`public/` の外にあるため）。編集する `status.json` だけが配信される。

---

## 今すぐ止める（3手・コピペ）

### 手順A: 全部止める（アプリ全体をブロック）

1. `public/litus/status.json` を開いて**中身を丸ごと下記に置き換えて保存**:

```json
{
  "schemaVersion": 1,
  "disabled": ["all"],
  "message": "大学からの要請により、リタスの提供を一時停止しています。再開までしばらくお待ちください。"
}
```

2. PowerShellで以下を丸ごと貼り付けて実行:

```powershell
cd C:\dev\waiteu-dev
git add public/litus/status.json
git commit -m "ops: リタスを緊急停止（all）"
git push origin master
```

3. **約30秒**でGitHub Actionsが自動デプロイ → 配信開始。以降、各端末は起動時と復帰時（最大15分間隔）に反映。

> `message` の文章は自由に書き換えてよい。空 `""` にすると、アプリ内の既定文言（「現在、リタスの提供を一時停止しています…」）が出る。

---

### 手順B: 一部の機能だけ止める

`disabled` に止めたい機能名を入れる（複数可）。機能名は **`attendance`（出席）/ `bulletin`（掲示）/ `letus`（課題）** の3つ。

例: 出席と課題だけ止める（掲示は使える）:

```json
{
  "schemaVersion": 1,
  "disabled": ["attendance", "letus"],
  "message": "出席・課題の機能を一時停止しています。"
}
```

保存後、手順Aの2（3つのgitコマンド）を同じように実行。コミットメッセージだけ内容に合わせて変える。

- 機能別停止では、その機能の画面に停止バナーが出て、収集（CLASS/LETUSへのアクセス）が止まる。アプリ自体は開ける。
- 全部止めたいときは手順A（`["all"]`）。

---

## 再開する（元に戻す）

1. `public/litus/status.json` を**通常運転の状態に戻して保存**:

```json
{
  "schemaVersion": 1,
  "disabled": [],
  "message": ""
}
```

2. PowerShellで:

```powershell
cd C:\dev\waiteu-dev
git add public/litus/status.json
git commit -m "ops: リタスを再開（通常運転）"
git push origin master
```

3. 約30秒でデプロイ。全停止中だったユーザーは、停止画面の「再確認」ボタンを押すか、アプリを開き直せば即復帰。放置でも次回起動／復帰（最大15分）で戻る。

---

## 反映の速さ（いつ効くか）

| 状況 | 反映タイミング |
|---|---|
| アプリを起動した瞬間 | すぐ反映（起動時に必ず確認） |
| アプリを開いている最中 | フォアグラウンド復帰のたびに確認・**最大15分に1回** |
| 全停止画面を見ているユーザー | 「再確認」ボタンで即・またはアプリ再起動で即 |

配信ファイルは `Cache-Control: no-store`（キャッシュさせない設定）なので、CDN側で古い値が残ることはない。**push→デプロイ完了（約30秒）が唯一の遅延**。

---

## 効いているか確認する

PowerShellで叩いて、今配信されている中身を確認:

```powershell
Invoke-WebRequest -Uri "https://waiteu.dev/litus/status.json?cb=$(Get-Random)" -Headers @{'Cache-Control'='no-cache'} -UseBasicParsing | Select-Object -ExpandProperty Content
```

デプロイ直後（数十秒）は古い値が返ることがある。10〜20秒あけて再実行。GitHub Actionsの進捗は https://github.com/waiteu-git/waiteu-dev/actions で見られる。

## スキーマ早見表

```json
{
  "schemaVersion": 1,          // 触らない（v1固定）
  "disabled": [ ... ],         // 停止対象の配列。空=通常運転
  "message": " ... ",          // 停止画面/バナーに出す本文。空=既定文言
  "title": " ... "             // （任意）全体停止画面の見出し。空/未指定=既定「リタスは一時停止中です」
}
```

**停止時の見せ方（`message` / `title`）は push するだけで随時変えられる**（アプリ再ビルド不要）。
- `message`: 全体停止画面の本文＋機能別バナーの文言に**そのまま**出る（機能別バナーは message を全文として表示。空なら「この機能は現在一時停止中のため…」）。
- `title`: 全体停止画面の見出しだけ差し替える（任意）。機能別バナーには見出しは無い。
- 例（全体停止で見出しも文面も差し替え）:
  ```json
  { "schemaVersion": 1, "disabled": ["all"], "title": "緊急メンテナンス中", "message": "システム更新のため一時停止しています。20分ほどで再開します。" }
  ```
- 注: `title` は **v76 以降のアプリ**で反映（それ以前のビルドは message のみ・title は無視）。

`disabled` に入れられる値:

| 値 | 効果 |
|---|---|
| `"all"` | アプリ全体を停止（ログイン前の起動画面でブロック。CLASS/LETUSに一切アクセスしない） |
| `"attendance"` | 出席機能を停止 |
| `"bulletin"` | 掲示機能を停止 |
| `"letus"` | 課題（LETUS）機能を停止 |

- 複数指定可。例 `["attendance", "bulletin"]`。
- 知らない値を入れても無視されるだけ（誤字は安全側に倒れる＝止まらない）。**止めたいときは綴りに注意**。

## 注意・安全設計

- **取得に失敗しても事故らない（fail-open）**: 端末が status.json を取れなかった場合、直近に取れた値を使う。一度も取れていない新規端末は通常動作。＝**サーバーが落ちてもアプリが道連れで止まることはない**。裏を返すと、**停止を効かせるには配信が生きている必要がある**（＝この仕組みが動く前提はwaiteu.devが生きていること）。
- **アップロードは一切ない**: アプリは status.json を**読むだけ**。端末情報・利用状況・個人情報を送る通信は存在しない（利用規約の「個人特定情報を送らない」と整合）。
- **JSONの書式に注意**: カンマ・引用符を壊すとアプリ側で「無効＝取得失敗」扱いになり、**停止が効かない**（fail-openのため通常動作に倒れる）。保存後は上の確認コマンドで実際の中身を必ず見る。
- **予約済みのローカル通知は停止中も鳴る**（大学システムには一切アクセスしないため許容。v1のスコープ外）。
